TDD

Test Driven Development

  1. Write tests
  2. Watch them fail/pass
  3. Refactor
  4. Repeat

BDD

Behaviour Driven Development

  • Establish goals
  • Use examples to describe the behaviour of the application
  • Automate those examples for quick feedback

Jasmine Syntax

describe("Simple multiplication", function() { it("multiples two numbers together", function() { expect(1 * 4).toEqual(4); }) });
